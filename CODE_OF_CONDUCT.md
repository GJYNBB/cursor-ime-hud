# Code of Conduct

## Our pledge

We want Cursor IME HUD to be a respectful, practical, and welcoming project for users and contributors. Please assume good faith, be patient with debugging information, and keep discussions focused on the project.

## Expected behavior

Examples of behavior that helps the project:

- Use welcoming and inclusive language.
- Be respectful of differing environments, workflows, and experience levels.
- Provide clear reproduction steps and diagnostics when reporting bugs.
- Accept constructive feedback gracefully.
- Keep feature discussions aligned with the project's narrow scope: showing IME state without reading user content or automatically switching input methods.

## Unacceptable behavior

Examples of unacceptable behavior include:

- Harassment, insults, threats, or personal attacks
- Publishing another person's private information without permission
- Repeatedly derailing issues or pull requests away from the project scope
- Posting exploit details or sensitive logs publicly instead of following [SECURITY.md](SECURITY.md)
- Other conduct that maintainers reasonably consider inappropriate in a professional open source project

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or pull requests; or restrict participation when behavior violates this Code of Conduct.

For security-sensitive reports, use the process in [SECURITY.md](SECURITY.md). For normal project support, use [SUPPORT.md](SUPPORT.md) and the GitHub issue templates.

This policy is based on the spirit of the [Contributor Covenant](https://www.contributor-covenant.org/).
